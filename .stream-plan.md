## Item 2 — `SubagentModel` live strip (structured wire signals only)

### Findings (real code) — which structured subagent signals reach the app at v29
- `SubagentModel` (`src/DaemonApp/Session/subagent_model.{h,cpp}`) consumes `{type:"subagent",
  id, title, status:"running"|"done"|"error", detail}` events via `applyEvents`. Fed **only** by
  the mock `TurnController` simulator (`turn_controller.cpp:47-57, 444-498`). `DaemonTurnEngine`
  emits **no** subagent events. 🔴 (strip is mock-only)
- `SessionOrchestrator` wires `ITurnEngine::eventsEmitted → m_subagents->applyEvents` and clears
  on turn end (`session_orchestrator.cpp:16-37, 235-241`). So any `{type:"subagent"}` the real
  engine emits will flow to the model with **zero orchestrator changes**.
- **`UserMsg.notice`** 🟢 — reaches `DaemonTurnEngine`'s Subscribe/LogPage stream today, already
  decoded as `DecodedLogEntry.hasNotice / noticeChild / noticeCallId` (`node_api_codec.h:330-339`)
  and already handled at `daemon_turn_engine.cpp:483-496` (emits a `delegationNotice` event). This
  is the node injecting a **detached/ephemeral child's completion** as a fresh reactive turn — a
  *Finished* signal only, and only for children the node re-injects.
- **`ManageEventView::Subagent`** 🟡 — the structured spawn/finish feed
  (`manage_event_view_subagent{ Subagent_seq, Subagent_child, Subagent_role,
  Subagent_phase: Spawned|Finished, Subagent_active_children }`,
  `daemon_api_client_types.h:3977-4014`). It rides `response_unit_events` (the `UnitEvents`
  reply). BUT: `encodeUnitEventsRequest` exists (`node_api_codec/fs_fleet_channels.cpp:313-320`)
  and is **never called anywhere**, and there is **no `decodeUnitEvents` facade** — so it does
  **not** reach the app today. `request_unit_events` has only `{unit, max}` (no cursor) →
  one-shot Call, not a cursored push stream.
- The session-log `agent_event_r` union (the turn stream) has **no** subagent/manage arm
  (`daemon_api_client_types.h:3419-3452`) — subagent phases never arrive on the transcript
  stream.
- Fleet tree `DecodedUnitNode` (role `EphemeralSubagent`/`ManagedChild`, `state`, `parentId`,
  `sessionId`, `title`, `work`, `lifetime`) 🟢 is decoded and kept **live** by `FleetRepository`
  off the `FleetChanged` feed arm (`subscription_manager.cpp:187-191`) — but it is fleet-scoped,
  not the focused turn engine. On the durable path `UnitId == SessionId`
  (`domain/unit_node.h:60`).

### Design — the exact structured-signal mapping into `SubagentModel`
Two structured sources feed the strip; the `DaemonTurnEngine` (owner of the focused session)
emits `{type:"subagent"}` events, `SubagentModel::applyEvents` upserts by `id`.

**Source A (primary — spawn + finish): `ManageEventView::Subagent` via `UnitEvents`.**
- New facade: `struct DecodedManageEvent { enum class Kind { Other, Started, Progress, Usage,
  Finished, Error, Subagent } kind; quint64 seq; QString child; QString role; QString phase; //
  "Spawned"|"Finished"; quint32 activeChildren; /* other arms' fields as needed, minimally */ };`
  and `static bool decodeUnitEvents(const QByteArray&, QList<DecodedManageEvent>* out);`
  (decode `response_unit_events`; append `ApiResponseKind::UnitEvents` mapping if not already —
  the enum value exists at `node_api_codec.h:788`, verify the reply-tag map row).
- `DaemonTurnEngine` (`daemon_turn_engine.{h,cpp}`): fetch `encodeUnitEventsRequest(m_sessionId,
  kUnitEventsMax)` (UnitId==SessionId) on turn start and re-fetch on activity (see cadence open
  question); decode via `decodeUnitEvents`; for each `Subagent` arm emit:
  - `Spawned` → `{type:"subagent", id: child, status:"running", title:<resolved>, detail:<role>}`
  - `Finished` → `{type:"subagent", id: child, status:"done", ...}`
  Track last-seen `seq` to avoid re-emitting already-applied events across re-fetches.

**Source B (completion linkage, already in stream): `UserMsg.notice`.**
- At `daemon_turn_engine.cpp:483-496`, in addition to the existing `delegationNotice` event,
  emit `{type:"subagent", id: noticeChild, status:"done", detail:<from callId link>}` when
  `entry.hasNotice`, so a detached child settles its strip row even if a `UnitEvents` `Finished`
  was missed. (`noticeCallId` links it to the parent `orchestrate` tool call.)

**Title / status caveats (structured-only → degradations):**
- `ManageEventView::Subagent` carries **no title/detail**. Title = child session id, upgraded to
  the roster/`SessionGet` title when available (cross-link to Item 1's `children`). Detail = role
  (or `active_children`).
- `Subagent_phase` is only `Spawned|Finished` — **no error phase**. So `status:"error"` cannot be
  derived from this source alone (would need the child `UnitNode.state==Finished` +
  `endReason`). `SubagentModel::failedCount()` therefore won't populate from Source A/B — see
  open questions (accept, or additionally consult the live fleet tree for error state).
- `active_children` (uint32) is a reliable live count for a strip header/badge.

**Absolutely NO** parsing of `notice`/display text to infer lifecycle — only the typed fields
above (`hasNotice`/`noticeChild` and the decoded `Subagent` arm).

### GUI + TUI parity
- Both surfaces already bind the **same** `SubagentModel`: GUI `SubagentList.qml` /
  `StatusStack.qml`; TUI `RootWidget::updateSubagents()` reads
  `m_active->orchestrator->subagents()` and also drives `StatusBarModel` agents-running/failed
  (`root_widget_strips.cpp:200-237`). So feeding the shared model lights up **both** with no
  per-surface logic. Verify `updateSubagents()` fires on the new events (it is triggered off the
  same eventsEmitted / `countChanged` path); wire a `countChanged`→`updateSubagents` connection
  in `root_widget_wiring.cpp` if not already present.

### Tests
- `tst_subagent_model.cpp`: already covers `applyEvents`; add cases for the new event shapes
  (spawn→running, finish→done, notice→done, re-emit dedupe by id).
- New codec test (in `tst_daemon_integration.cpp` or a fleet test): hand-encode a `UnitEvents`
  response with a `Subagent` arm; assert `decodeUnitEvents` maps child/role/phase/activeChildren.
- `tst_session_orchestrator.cpp`: assert the real-engine path (fake events) reaches
  `subagents()` (running/failed/count).
- TUI: extend a `tests/tui` case (parity) to assert the footer/`updateSubagents` strip renders
  the live rows.

## 1. Architecture & shared seams (decisions)

### 1.1 The auth client seam (Part 1) — what exists vs what's needed
- `node_api_auth.{h,cpp}` is **SASL connection auth** (SCRAM/PLAIN/EXTERNAL for the socket
  handshake) — *not* interactive `AuthBegin/AuthComplete`. Do **not** touch it.
- `IAccountsService::beginOAuth` is the only "OAuth" seam today; `MockAccountsService` **simulates**
  it and `DaemonAccountsService::beginOAuth` reports `oauthFailed` (unimplemented). This seam is
  account-centric and returns nothing usable for a flow URL. **We will not overload it.**
- **Decision:** add a dedicated, thin **`IAuthFlowService`** interface + a shared **`AuthFlowController`
  QObject view-model** (GUI+TUI bind the same instance), a daemon **`AuthRepository`** (the wire
  seam), a **`MockAuthFlowService`** for offscreen/mock, and a small **redirect-capture** helper
  (loopback `QTcpServer`). This keeps auth op-names isolated to `AuthRepository`+`NodeApiCodec`
  (the wire-rename requirement).

### 1.2 Wire-rename isolation (Acp* → Agent* + protocol field, next version)
- All `Acp*`/`Auth*` op-name string literals stay **inside `NodeApiCodec` (encode/decode) and the
  repository layer** (`AcpRepository`, new `AuthRepository`). QML/TUI call neutral methods
  (`AcpAgents.discover()`, `AuthFlow.begin(...)`) and never name a wire op. When the rename lands,
  only the codec's request-choice member names + the two repos change; no QML/TUI edits.
- The QML singleton is already `AcpAgents` (agent-neutral). Keep it. Add `discover()` as a neutral
  verb; the `AcpDiscover` op string lives only in `encodeAcpDiscoverRequest`.

### 1.3 GUI+TUI parity (mandatory, per `daemon-app/AGENTS.md`)
- Every GUI surface below has a named TUI counterpart. Shared *state/behavior* lives in C++ view
  models (`AuthFlowController`, `FirstRunModel`, `AcpRepository`); QML and `src/tui` are renderers.
  The auth "sheet" is a graphical hand-off; the TUI reduced form = **copy-URL + status/poll panel +
  manual-callback paste** (explicitly a reduced form, stated here).

## 2. Part 1 — shared `AuthFlowSheet` (generic `AuthBegin`/`AuthComplete`)

### 2.1 Codec facade additions — `src/core/daemon/node_api_codec.{h,cpp}`
New decoded structs (in the header, beside the existing `Decoded*`):
```cpp
struct DecodedAuthProviderInfo { QString family; QString flowKind; QString displayName;
                                 QList<DecodedAuthParamField> params; };   // params_schema
struct DecodedAuthParamField   { QString key; QString label; bool required=false; bool secret=false; };
struct DecodedAuthBeginResponse{ QString flowId; QString authorizationUrl; QString redirectUri;
                                 quint64 expiresAt=0; QString flowKind; };
struct DecodedAuthCompleteResponse{ QString credentialRef; QString accountLabel;
                                    QString transportInstance; bool hasBoundProfile=false;
                                    QString boundProfile; };
```
New encoders/decoders + `ApiResponseKind::{AuthProviders, AuthBegun, AuthCompleted}`:
```cpp
static QByteArray encodeAuthProvidersRequest();
static QByteArray encodeAuthBeginRequest(const QString& family, const QVariantMap& params,
                                         const QString& redirectUri,
                                         const QString& bindProfile = {},        // bind.profile
                                         const QString& bindCredentialRef = {}); // bind.credential_ref
static QByteArray encodeAuthCompleteRequest(const QString& flowId, const QString& callback);
static QByteArray encodeAuthCancelRequest(const QString& flowId);
static bool decodeAuthProviders(const QByteArray&, QList<DecodedAuthProviderInfo>* out);
static bool decodeAuthBegun(const QByteArray&, DecodedAuthBeginResponse* out);
static bool decodeAuthCompleted(const QByteArray&, DecodedAuthCompleteResponse* out);
```
Note the `params` map is CBOR `map<tstr,tstr>` (`auth_begin_request_params_tstrtstr_*`), so
`QVariantMap` → string/string. Extend `responseKind()` switch to map the three new arms.

### 2.2 Wire seam — `src/core/daemon/repositories.{h,cpp}` (new `AuthRepository`)
Mirror the existing repository idiom (correlation-id table, `handleResponse`/`handleFailure`):
```cpp
class AuthRepository : public RepositoryBase {
  Q_OBJECT
public:
  void refreshProviders();                                   // AuthProviders
  void begin(const QString& family, const QVariantMap& params,
             const QString& redirectUri, const QString& bindProfile = {});   // AuthBegin
  void complete(const QString& flowId, const QString& callback);             // AuthComplete
  void cancel(const QString& flowId);                                        // AuthCancel
  [[nodiscard]] const QList<DecodedAuthProviderInfo>& providers() const;
signals:
  void providersRefreshed();
  void begun(const DecodedAuthBeginResponse&);
  void completed(const DecodedAuthCompleteResponse&);
  void failed(const QString& phase, const QString& message);  // "begin"|"complete"|"providers"
};
```
Correlations: `repo/auth-providers`, `repo/auth-begin`, `repo/auth-complete`, `repo/auth-cancel`.

### 2.3 Client interface + view-model — new `src/core/auth/`
- `iauth_flow_service.h` — thin interface the controller depends on (so mock + daemon both satisfy):
  `providers()`, `begin(family, params, redirectUri, bindProfile)`, `complete(flowId, callback)`,
  `cancel(flowId)`, signals mirroring `AuthRepository`.
- `auth_flow_controller.{h,cpp}` — the **shared GUI+TUI view-model** (`QObject`, Q_PROPERTY-driven):
  ```
  Q_PROPERTY phase            // "idle"|"beginning"|"awaiting_browser"|"completing"|"success"|"failed"|"cancelled"
  Q_PROPERTY authorizationUrl // copyable / open-in-browser
  Q_PROPERTY flowKind         // "MatrixSso"|"OAuth2Pkce" (label only)
  Q_PROPERTY error
  Q_PROPERTY accountLabel     // on success
  Q_INVOKABLE start(family, params)  // allocates redirect sink, calls begin()
  Q_INVOKABLE openInBrowser()        // Qt.openUrlExternally(authorizationUrl) (GUI)
  Q_INVOKABLE submitCallback(cb)     // manual paste path (TUI / non-loopback)
  Q_INVOKABLE cancel()
  signals: succeeded(credentialRef, accountLabel, transportInstance), failed(reason), phaseChanged...
  ```
  State machine: `start()` → open loopback sink (2.4) → `service->begin(family, params, redirectUri)`
  → on `begun` store url + `phase=awaiting_browser` (auto-`openInBrowser()` on GUI) → sink captures
  redirect **or** user pastes callback → `service->complete(flowId, callback)` → `phase=completing`
  → on `completed` `phase=success` + emit `succeeded`; on any `failed` → `phase=failed`,
  `cancel(flowId)` best-effort. TTL from `expiresAt` drives a timeout → failed.
- `redirect_sink.{h,cpp}` — a short-lived loopback `QTcpServer` bound to `127.0.0.1:0`; yields
  `http://127.0.0.1:<port>/cb` as `redirectUri`, serves a "you may close this tab" page on the GET,
  extracts the raw query string, emits `captured(callback)`, tears down. Guarded so headless/TUI can
  skip it and use `submitCallback()` instead. (Matches auth-spec §8 desktop shape.)
- `mock_auth_flow_service.{h,cpp}` — canned begin→complete for offscreen tests (no socket, no
  browser): `begin` returns a fake url + flow_id; `complete` echoes a `credential_ref`. Lets the
  controller state machine + QML/TUI be tested deterministically.

## A1 decision outcome (coordinator question 3)
**`ProviderRegister` is NOT used.** Read `daemon-host`/`daemon-api` dispatch: the only
`provider_register` implementation is the `ControlApi` default returning
`ApiError::Unsupported("provider_register")` — no node override exists, so the op cannot serve a
probe or persist anything. The landed path: profile `base_url` + **`provider = "daemon_api"`**
(not the coordinator-suggested `genai`: per `bins/daemon/src/main.rs`, `GenAi` infers the vendor
wire from the MODEL NAME — `claude-*` against a vLLM server would get Anthropic framing — while
`DaemonApi` is genai's OpenAI-compatible adapter pinned at `spec.base_url`, exactly the custom
self-hosted case) + user-typed model + optional profile-scoped key. **No inline pre-commit
reachability probe is possible**: `ProviderModels` carries no base-URL arg, so no wire op can
list an arbitrary endpoint. The UI says so honestly ("your first message verifies it…"), the A7
probe deliberately skips descriptor-less providers, and CON-8's send-time re-entry is the
failure path. A future node capability (e.g. `ProviderModels` gaining `base_url`, or a real
`provider_register`) would make the probe honest — noted for integration.

## Deviations from the Phase A plan
- **No `tests/qml/auth_flow_sheet` / `tests/qml/agent_type_picker` harnesses.** The QML-level
  behavior is thin (phase-driven visibility) over controllers that are unit-tested
  (`tst_auth_flow`, `tst_first_run_model`); the offscreen QML harness cost was spent on the
  shared-model tests instead, matching the AGENTS.md guidance that shared-model coverage lives
  in `tests/unit`. Existing `tests/qml/downloads_panel` keeps covering the touched panel.
- **No separate `tst_acp_repository` / `tst_model_quantize` files**: the AcpDiscover +
  quantize/inspect round-trips live in `tst_auth_flow.cpp` (one stream-owned TU), and
  `AcpRepository::discover()` reuses the already-covered decode path.
- **TUI CON-8 reduced form**: the TUI renders the node's actionable error inline (pre-existing
  `composer_chrome` line) and re-raises the first-run gate on done→active transitions
  (`reenterProvider()`/settings re-run); it does not add a dedicated "Set up provider" keybind.
  Stated degradation per the parity policy.
- **A6 probe trigger**: the ACTIVE local model is probed on every `currentRefreshed` (bounded,
  meaningful); rows are NOT probed en-masse per catalog rebuild (N round-trips). On-demand
  `verifyInstalled(id)` exists on the facade for future surfaces.
- **A4 free-disk**: per coordinator decision, size-estimate only; the preflight line documents
  the future `disk_free` slot.
- **`AuthFlowSheet` gained an idle family-picker state** (schema-driven params from
  `AuthProviders.params_schema`) so the sheet is self-serve for any consumer (AccountsPage
  today, the sibling Channels stream tomorrow) — openFlow(family, params) still skips it.
- **TTL edge hardened**: an expires_at already in the past on arrival fails closed immediately
  (was: silently never expiring).
